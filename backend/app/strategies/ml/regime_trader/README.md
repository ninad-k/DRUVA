# Regime-Trader: HMM-Based Volatility Regime Detection

A production-grade volatility regime classifier built for **DRUVA**.

## Overview

**Regime-Trader** uses Hidden Markov Models (HMM) to detect market volatility regimes in real-time. Unlike trend-following strategies, it focuses on *regime state*, allowing position sizing to adapt dynamically to market conditions.

### Regimes (5-state model)

| ID | Name | Characteristics | Recommended Allocation | Leverage |
|---|---|---|---|---|
| 0 | **Crash** | Extreme volatility, sharp drawdown | 5% | 1.0x |
| 1 | **Bear** | High volatility, downtrend | 30% | 1.0x |
| 2 | **Neutral** | Medium volatility, sideways | 50% | 1.0x |
| 3 | **Bull** | Low volatility, uptrend | 95% | 1.25x |
| 4 | **Euphoria** | Low volatility, extreme upside | 80% | 1.0x |

## Features

### 1. **Forward Algorithm (No Look-Ahead Bias)**
- Uses HMM forward inference, not Viterbi
- Each prediction depends only on past data
- Safe for live trading without information leakage

### 2. **Stability Filter**
- **Persistence**: Requires 3 consecutive bars in same regime before signal
- **Flickering Detection**: Warns if regime switches > 4 times in 20 bars
- Prevents whipsaw trades from noisy regimes

### 3. **Dynamic Allocation**
- Position sizing adapts to regime
- Bull: 95% allocation + 1.25x leverage → aggressive
- Crash: 5% allocation, 1.0x leverage → defensive
- Customizable via strategy parameters

### 4. **Technical Features**
Engineered from OHLCV data:
- **Returns**: 1-bar, 5-bar, 20-bar log returns
- **Volatility**: 20-bar rolling standard deviation
- **Volume**: Log of 20-bar rolling average volume
- **Preprocessing**: StandardScaler normalization

## Usage

### 1. **Train the HMM Model**

```python
import pandas as pd
from app.strategies.ml.regime_trader.hmm_engine import RegimeDetector

# Load your historical OHLCV data
ohlcv = pd.read_csv("data.csv", index_col="timestamp")

# Fit HMM on at least 1 year (252 days) of data
detector = RegimeDetector(n_regimes=5)
detector.fit(ohlcv)

# Save trained model
detector.save("backend/app/strategies/ml/regime_trader/models/v1/model.pkl")
```

### 2. **Enable the Strategy in DRUVA**

Create a strategy config in your database or JSON:

```json
{
  "id": "regime_trader_main",
  "strategy_name": "regime_trader.hmm.v1",
  "enabled": true,
  "parameters": {
    "allocations": {
      "0": 0.05,   // Crash: 5%
      "1": 0.30,   // Bear: 30%
      "2": 0.50,   // Neutral: 50%
      "3": 0.95,   // Bull: 95%
      "4": 0.80    // Euphoria: 80%
    },
    "leverage": {
      "0": 1.0,
      "1": 1.0,
      "2": 1.0,
      "3": 1.25,   // 1.25x in Bull
      "4": 1.0
    },
    "confidence_threshold": 0.60,  // Min confidence to change regime
    "max_flickers": 4              // Max regime switches per 20 bars
  }
}
```

### 3. **Signals & Interpretation**

The strategy emits allocation signals, not buy/sell signals:

| Signal | Meaning | Action |
|--------|---------|--------|
| **BUY** | Confident uptrendregime (Bull/Euphoria, ≥3 bars) | Increase exposure to current allocation |
| **SELL** | Confident downtrendregime (Crash/Bear, ≥3 bars) | Reduce exposure / increase hedges |
| **HOLD** | Uncertain or neutral regime | Maintain current positions |

**Metadata** (returned in every prediction):
```python
meta = {
    "regime_name": "Bull",              # Human-readable regime
    "regime_id": 3,                     # Integer [0-4]
    "confidence": 0.78,                 # Model confidence [0-1]
    "allocation_pct": 95.0,             # Recommended portfolio %
    "leverage": 1.25,                   # Recommended leverage
    "persistence_bars": 5,              # Bars in current regime
    "flicker_warning": False,           # True if > 4 switches/20 bars
}
```

## Integration with Risk Manager

The Risk Manager uses these signals to enforce circuit breakers:

```python
# Risk Manager receives regime allocation from prediction.meta
if daily_pnl <= -2% * equity:
    # Circuit breaker: cut all positions by 50%
    position_max_size = regime_allocation * 0.5

if daily_pnl <= -3% * equity:
    # Circuit breaker: close all positions
    close_all_positions()

if drawdown_from_peak >= 10%:
    # Kill switch: lock bot, require manual restart
    write_lock_file()
```

## Configuration Tuning

### Allocations

Adjust based on your risk tolerance:
- **Conservative**: Crash 5%, Bull 50% (low leverage)
- **Moderate**: Crash 5%, Bull 95% (default, 1.25x in Bull)
- **Aggressive**: Crash 10%, Bull 95%, Euphoria 100% (high leverage)

### Confidence Threshold

- **0.55**: Fast regime switches, more frequent trades
- **0.60**: Balanced (default)
- **0.70**: Slower, fewer false signals

### Flickering Tolerance

- **2**: Strict, less tolerance for noise
- **4**: Default, balanced
- **6+**: Very relaxed, may miss reversal
